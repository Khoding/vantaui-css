# VantaUI CSS: Data Display

> Surfaces and data components: panels, tables, description lists, badges, avatars, key/value & stat readouts, timelines, steppers, trees, carousels, ratings, and segmented controls.

Prerequisites: opt in with `class="vui"` on a root element. Bare semantic HTML is styled with NO prefix; helpers are single element-scoped words. A leading `<i>text</i>` is a Material Symbols icon. Mark the current/selected item `aria-current="page"` (alias `.active`). See the hub: https://vantaui-css.khodok.com/llms.txt

Related: alerts/toasts/meters/spinners → https://vantaui-css.khodok.com/llms-feedback.txt · color tokens for tones → https://vantaui-css.khodok.com/llms-theming.txt

## Panel (`<article>`)

A bare `<article>` is a chamfered plate. A `<header>`/`<footer>` inside it auto-spans edge-to-edge with a hairline divider (detected via `:has`). A trailing `<small>` in the header is a meta note.

- **Depth**: `raised` · `inset` · `flat` (no shadow).
- **Stage**: `stage` recesses the body into a dark ink + scanlines viewport while any `<header>`/`<footer>` rise to the panel surface and frame it — the framed-console / readout look (e.g. `<article class="stage">`).
- **Shape**: `notch` (all corners cut) · `brackets` (glowing corner ticks).
- **Glow**: `glow` (accent rim glow).
- **Color tint**: `cyan`, `amber`, `red`, or `green` (these are admonition tones that tint the surface, recolor the text, and set the panel's accent color so that child buttons inherit the same tone).
- **Tint intensity** (with a color): `dim` · `vivid`.
- **Linked / tappable**: `linked` makes the hover action shift the border to the accent color and raise the background. Use this on panels that are wrapped in or function as links.
- Combine freely: `"amber glow"`, `"green raised"`, `"red notch brackets"`, `"cyan vivid linked"`.

```html
<article class="raised">
  <header><small class="vui-eyebrow">Diagnostics</small> <small>v2.1</small></header>
  <p>A chamfered panel; the header drew its own divider.</p>
  <footer><button type="submit">Authorize</button></footer>
</article>
<article class="amber glow"><strong>Caution</strong><p>Tinted admonition panel.</p></article>
```

(Note: `<article class="vui-prose">` or `<article class="prose">` opts out of panel styling and uses the long-form prose layout documented in the layout file.)

## Table

A bare `<table>` is a mono data grid with eyebrow (uppercase) headers and cyan row hover. Use real `<thead>`/`<tbody>`/`<th>`/`<td>`. Wide tables self-scroll rather than breaking the layout.

```html
<table>
  <thead><tr><th>Unit</th><th>Status</th><th>Integrity</th></tr></thead>
  <tbody>
    <tr><td>Raven-1</td><td>Online</td><td>98%</td></tr>
  </tbody>
</table>
```

## Description list (`<dl>`) as a telemetry readout

A bare `<dl>` is a label │ value grid: each `<dt>` an uppercase label, each `<dd>` a mono value, divided by hairlines. Add `.stacked` for label-over-value rows. Pairs naturally inside a panel.

```html
<dl>
  <dt>Callsign</dt><dd>RAVEN-1</dd>
  <dt>Sector</dt><dd>NE-04</dd>
</dl>
```

## Key/value row & big stat

- **`.kv`**: a single key/value readout row (with the label on the left, a monospaced value on the right, and a hairline border underneath). This is useful for ad-hoc rows where a full `<dl>` is unnecessary.
- **`.stat`**: a large numeric readout. Use `<b>` for the large HUD number and `<span>` for the uppercase caption. You can use `.stat.signal` to light the number in cyan, or `.stat.amber` for amber.

```html
<div class="kv"><span>Uplink</span><span>42 ms</span></div>
<div class="stat signal"><b>98.6</b><span>Integrity</span></div>
```

## Badge / status tag

A `.badge` is a status chip (on a `<span>`, `<a>`, `<mark>`…). Tone: `cyan` · `amber` · `red` · `green` · `neutral`. Add `.solid` to fill, `.dot` for a leading live dot. A leading `<i>`/`<svg>` sits inline; long labels truncate with an ellipsis.

```html
<span class="badge green dot">Online</span>
<span class="badge red solid">Threat</span>
<span class="badge amber"><i>warning</i> Degraded</span>
```

## Avatar

Operator identity on a `<span>`/`<div>` (initials) or an `<img>`. A chamfered plate by default.

- **Shape**: `round` (disc) · `hex` (six-sided).
- **Size**: `small` · `large`.
- **Status frame** (glowing): `online` · `busy` · `idle` · `offline`.
- **`.avatar-group`** overlaps several into a stack.

```html
<span class="avatar online">R1</span>
<img class="avatar round large" src="op.jpg" alt="Raven-1" />
<div class="avatar-group">
  <span class="avatar">R1</span><span class="avatar">B2</span><span class="avatar">+5</span>
</div>
```

## Timeline / telemetry log

An `<ol class="timeline">` is a vertical event feed: each `<li>` gets a glowing node on a connecting rail. Lead an `<li>` with a `<time>` for the stamp; tone a node with a color word on the `<li>` (`cyan` · `amber` · `red` · `green`). Add `.log` for a dense, monospaced feed.

```html
<ol class="timeline">
  <li class="green"><time>09:42:01</time> Uplink established.</li>
  <li class="amber"><time>09:43:18</time> Signal degraded.</li>
  <li class="red"><time>09:45:00</time> Link lost.</li>
</ol>
```

## Stepper / wizard progress

An `<ol class="stepper">` represents steps with numbered nodes joined by a connector line. Mark the active step with `aria-current="step"`, and add `.done` to a completed step to replace its number with a checkmark. You can also add `.vertical` for a vertical layout. Since the stepper is static by design, you will need to handle navigation logic in your application by toggling the `.done` class and the `aria-current` attribute.

```html
<ol class="stepper">
  <li class="done">Authenticate</li>
  <li aria-current="step">Configure</li>
  <li>Deploy</li>
</ol>
```

## Tree (disclosure hierarchy)

A `<ul class="tree">` displays a hierarchy of rows with guide rails. A branch is structured as `<li><details><summary>…</summary><ul>…</ul></details></li>`. This collapses automatically with no JavaScript and animates open on browsers that support `::details-content`. A leaf is a plain `<li><a>` or `<li><span>`. You can lead any row with an `<i>` folder or file icon, and mark the selected leaf with `aria-current="page"` or `.active`.

```html
<ul class="tree">
  <li><details open>
    <summary><i>folder</i> Sector NE</summary>
    <ul>
      <li><a aria-current="page"><i>description</i> Brief.md</a></li>
      <li><a><i>description</i> Map.png</a></li>
    </ul>
  </details></li>
  <li><a><i>folder</i> Sector SW</a></li>
</ul>
```

For a full ARIA tree widget (roving tabindex, type-ahead, arrow-key walk) add `role="tree"`/`role="treeitem"`/`role="group"` + `aria-expanded` and drive focus in JS.

## Carousel (scroll-snap filmstrip)

A `<div class="carousel">` is a horizontal scroll-snap track where the direct child elements are the slides. It supports swipe, scroll, and keyboard navigation by default. On browsers that support native CSS carousels, it automatically adds navigation arrows and a row of snap-marker dots without requiring any JavaScript or external libraries.

- Size slides with `--vui-slide`.
- `.peek` shows a sliver of the next slide · `.full` snaps one slide per view · `.clean` hides the scrollbar · `.no-arrows` / `.no-dots` drop either native control.

```html
<div class="carousel peek" style="--vui-slide: 18rem">
  <article>Slide 1</article>
  <article>Slide 2</article>
  <article>Slide 3</article>
</div>
```

## Rating / signal-strength

A `<fieldset class="rating">` renders as a row of pips that fill up to the selected value when clicked or navigated with arrow keys. Each pip consists of a `<label>` (which uses a leading `<i>` for its icon) immediately followed by its radio `<input>`, ordered from lowest to highest value (1 to 5). The `<legend>` serves as the group label. You can tone the control using a color word like `cyan`, `amber`, `red`, or `green`. Add the `.readonly` class or disable the inputs to make it a static readout. This component requires no JavaScript since it uses real radios and is styled using `:has()`.

```html
<fieldset class="rating amber">
  <legend>Signal</legend>
  <label><i>signal_cellular_alt</i></label><input type="radio" name="sig" value="1" />
  <label><i>signal_cellular_alt</i></label><input type="radio" name="sig" value="2" checked />
  <label><i>signal_cellular_alt</i></label><input type="radio" name="sig" value="3" />
</fieldset>
```

## Segmented control

A segmented control acts as a single-select switch. It is structured as a `.segmented` track of `<label>` chips, each wrapping a radio `<input>` and an optional leading `<i>`. The checked segment is highlighted automatically. This control is built using real radios, meaning it works without JavaScript and supports native keyboard navigation. You can size it with `.small` and tone it with a color word like `amber`, `red`, or `green` (defaulting to cyan).

```html
<div class="segmented">
  <label><input type="radio" name="view" checked /> Map</label>
  <label><input type="radio" name="view" /> List</label>
  <label><input type="radio" name="view" /> Grid</label>
</div>
```
