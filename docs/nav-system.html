Chrome & Navigation

Write the landmarks, get the layout. A <header> with a <nav> becomes the app bar; a <footer> becomes the footer; a <dialog class="left"> is an off-canvas drawer; a .vui element holding a <main> becomes the whole app frame. One-word helpers tune the rest. Resize the window and watch the bar above collapse into the burger — that's a container query on the header itself.

Header

A <header> with a <nav> is an app bar. The first link is the brand; the first nav pushes the trailing <menu> to the end — no spacer element. Shapes are one word: glow · center · float · bare · tall.

Navigation

Four ways to move around, all from semantics: an off-canvas drawer, a phone command bar, a breadcrumb, and the side rail (shown in the App Shell below).

Off-canvas drawer

A native <dialog class="left"> (or right). Opens with showModal(); animates in with @starting-style + allow-discrete. Esc and backdrop-click close it.

App Shell

Any .vui element that directly holds a <main> becomes an app frame and places its landmarks by element — no wrapper classes. Add a <nav class="left"> and it grows a side rail (a vertical column ≥768px, a bottom bar below). On <body> it fills the viewport; here it's scaled inside a box.
