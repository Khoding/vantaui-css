# VantaUI CSS: Theming & Design Tokens

> Every design token (CSS custom property) and how to override it: the OKLCH color system, surfaces, text, status, spacing/geometry, typography, effects, motion, and keyframes. Authoritative as of v1.7.1.

Prerequisites: design tokens are defined on `:root` and are available everywhere, including outside the `.vui` scope. Outside `.vui`, only tokens are defined, meaning nothing else changes. You can override any token anywhere because they cascade. See the hub: https://vantaui-css.khodok.com/llms.txt

Related: type-role/utility classes that consume these tokens → https://vantaui-css.khodok.com/llms-layout.txt

## How theming works

VantaUI is token-driven. To re-skin the interface, override the custom properties instead of touching component CSS. The primary accent color is electric cyan (`--accent`). If you change this token, all buttons, focus rings, glows, links, and active states will update to match.

```css
:root {
  --accent: oklch(84.6% 0.133 212.1); /* the signal color */
  --bevel-md: 14px;                   /* deeper chamfers everywhere */
  --container-max: 90rem;
  --font-display: 'Chakra Petch', sans-serif;
}
```

Scope a theme to a subtree by setting tokens on a container instead of `:root`. Panel color variants already remap their accent locally so nested buttons inherit the tone (see the data file).

## Colors (OKLCH)

Colors are OKLCH. Every token ships twice in `src/`: an sRGB hex/rgba fallback first, then the canonical `oklch()` value (engines without `oklch()` keep the fallback; they're the same color). The `dist` build targets OKLCH-capable engines. Palette meaning: cyan = you / system, amber = caution / hostile, red = threat, green = secure / online.

- **Color scales**: `--ink-{1000,900,800,700,600,500,400}` (surfaces, cool near-black). `--cyan-{50,200,400,500,600,700}` + `--cyan-glow` (400 is the primary accent). `--amber-{300,400,500,600}` + `--amber-glow`. `--red-{400,500,600}` + `--red-glow`. `--green-{400,500,600}` + `--green-glow`. `--slate-{50,200,300,400,500,600}` (neutrals / text).
- **Backgrounds**: `--bg-{void,base,raised}`, `--surface-{panel,panel-2,inset}`.
- **Text**: `--text-{primary,secondary,muted,faint,signal,invert}`.
- **Accent / brand**: `--accent` `--accent-strong` `--accent-dim` `--accent-glow`.
- **Status**: `--status-{online,warn,threat,hostile}`.
- **Lines / overlays**: `--hairline` `--hairline-strong` `--hairline-accent` `--divider` `--scrim`.

## Spacing & geometry

- **Spacing scale**: `--space-0…--space-13`. (0=0, 1=2px, 2=4px, 3=8px, 4=12px, 5=16px, 6=20px, 7=24px, 8=32px, then up to 13.)
- **Radius**: `--radius-{none,xs,sm,pill}`.
- **Bevel / chamfer cut sizes**: `--bevel-{sm,md,lg,xl}`. Ready clip-paths include `--clip-chamfer` (cut top-left and bottom-right) and `--clip-notch` (all four corners cut). Every beveled component progressively upgrades from clip-path to `border-radius` and `corner-shape: bevel` on compatible browsers (Chromium 139+) — `<article>` panels and `.vui-panel`, `.vui-chamfer`/`.vui-notch`, `<pre>`, and the rest of the library: buttons, badges, inputs/select/textarea/checkbox, tables, menus & dropdowns, dialogs & tooltips, tabs, alerts, toasts, avatars, toolbars, segmented controls, pagination, steppers, skeletons, and carousel controls. This offers an identical look, but `corner-shape` only shapes the box boundary without clipping overflowing text or dropdown menus the way `clip-path` does. On other browsers (Firefox/Safari as of 2026), `clip-path` remains the fallback. You can add the `.vui-clip` class to any such element to opt out of this upgrade. This keeps the clip-path version and restores the open, unstroked diagonal corner (the classic "snipped" HUD look), though it will clip overflowing content (which you must then manage yourself). One-sided diagonals — the drawer trailing edge, meter/progress bars, the range/switch thumbs — are not corner bevels and stay on clip-path everywhere.
- **Borders**: `--border-{hair,thin,bold}`.
- **Container / frame**: `--container-max` `--container-narrow` `--container-pad` `--gutter` `--hud-edge` `--drawer-width`.
- **Breakpoints**: `--bp-{s,m,l,xl}`.

## Typography

- **Families**: `--font-{display,ui,hud,mono,icon}` (Chakra Petch · Rajdhani · Share Tech Mono · Material Symbols).
- **Weights**: `--fw-{light,regular,medium,semibold,bold}`.
- **Fluid sizes**: `--fs-{2xs,xs,sm,base,md,lg,xl,2xl,3xl,4xl,5xl}` (base and up use `clamp()`; size using these rather than media queries).
- **Line height**: `--lh-{tight,snug,normal,mono}`. **Letter-spacing**: `--ls-{tight,normal,wide,wider,widest}`.
- **Semantic sizes**: `--text-{display,title,heading,body,label,mono}-size`.

## Effects & motion

- **Glows**: `--glow-cyan-{sm,md,lg}` `--glow-{amber,red,green}-md`. **Text glow**: `--text-glow-{cyan,amber}`.
- **Shadows**: `--shadow-{panel,raised,inset}`.
- **Textures**: `--scanlines` `--grid-fine` (+`--grid-fine-size`) `--vignette`.
- **Motion**: `--ease-{hud,snap,inout}` `--dur-{fast,base,slow,boot}`.
- **Focus**: `--focus-ring`.

## Keyframes

Available animations: `vui-flicker`, `vui-scan`, `vui-pulse`, `vui-sweep` (and the component-internal `vui-skeleton`, `vui-toast-in`/`vui-toast-out`). A global `prefers-reduced-motion: reduce` rule neutralizes ambient animation.

## Fonts loading

Fonts (Chakra Petch, Rajdhani, Share Tech Mono, and Material Symbols) load automatically via `@import` inside the stylesheet. Because the remote `@import` can break some build tools (like LightningCSS), you can vendor a copy of the stylesheet with the `@import` line removed and load the fonts separately.

### Self-hosting the fonts (recommended for production)

The bundled `@import` is convenience, not a requirement. Self-hosting the four families is faster (no extra connection to `fonts.googleapis.com`, no render-blocking remote `@import`, fonts cached on your own origin) and keeps visitor requests off Google's CDN.

If your framework has a font manager, let it handle this. For example, **[Nuxt Fonts](https://fonts.nuxt.com/)** or **[`@nuxtjs/google-fonts`](https://google-fonts.nuxtjs.org/)** will download, subset, and self-host the fonts automatically. You only need to declare the four families and remove the remote `@import` line.

Without a font manager, do it by hand:

1. Open each family on Google Fonts: [Chakra Petch](https://fonts.google.com/specimen/Chakra+Petch), [Rajdhani](https://fonts.google.com/specimen/Rajdhani), [Share Tech Mono](https://fonts.google.com/specimen/Share+Tech+Mono), [Material Symbols](https://fonts.google.com/icons), and download the font files (or grab the `@font-face` CSS that the Google Fonts URL returns).
2. Drop the files under e.g. `/fonts/` in your app and serve them from your own origin.
3. Strip the remote `@import` from the VantaUI stylesheet (vendor a copy with that line removed) and load your local `@font-face` declarations instead. The token family names (`--font-display` etc.) stay the same, so nothing else changes.

Chakra Petch, Rajdhani, and Share Tech Mono are licensed under the SIL Open Font License 1.1, while Material Symbols is licensed under the Apache License 2.0. When self-hosting the actual files, please include those license texts alongside them.
