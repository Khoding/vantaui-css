# VantaUI CSS: Optional JavaScript & Framework Integration

> The optional behaviours module (tabs, animated meters, drawers, dropdown menus, toolbars, tooltips, live clock, toasts) and the Vue 3 plugin + Nuxt 3 module. The CSS is fully usable with no JavaScript, as the scripts only add interactive enhancements. Authoritative as of v1.6.1.

Prerequisites: the CSS works on its own. Load the JS only for the interactions below. See the hub: https://vantaui-css.khodok.com/llms.txt

Related: the markup these behaviours enhance → https://vantaui-css.khodok.com/llms-navigation.txt and https://vantaui-css.khodok.com/llms-feedback.txt

## Install / boot

```js
import { init } from 'vantaui-css/js';
init(); // boots tabs, animated meters, drawers, menus, toolbars, tooltips, clock
```

Works as an ES module or a plain `<script>` (then it's attached to `window.vui` and auto-inits on `DOMContentLoaded`). All functions take an optional `root` (default `document`) so you can re-run them on dynamically added markup. Most are idempotent (guarded against double-wiring).

Package entry points (`exports`): `vantaui-css/js` (ES module), `vantaui-css/js/global` (plain-script global build).

## API

- **`init(root = document)`**: runs all behaviours below.
- **`tabs(root)`**: wires `[role="tablist"]`/`[role="tab"]` (click and Left/Right arrow keys). Panels are matched by the tab's `aria-controls` or `data-tab` attribute to the matching element ID, toggles the `hidden` attribute, and sets `aria-selected` and `tabindex`. It activates the `aria-selected="true"` tab, defaulting to the first tab if none is selected.
- **`setValue(el, n, duration = 600)`**: animates a `.meter` or `.gauge`'s `--value` using an ease-out cubic curve (respecting `prefers-reduced-motion` and `duration <= 0`). It reads the maximum value from `--max` or `data-max` (defaulting to 100) and clamps the result to `[0, max]`.
- **Auto-animate (via `init`)**: elements with both `data-value` and `data-animate` fill from 0 when scrolled into view (using an IntersectionObserver with a threshold of 0.4, falling back to an immediate fill if the observer is unavailable).
- **`drawers(root)`**: registers a global click handler where `[data-open="id"]` opens a `<dialog id>` using `showModal()` (or toggles `.active` or `.open` on a non-dialog drawer), `[data-close]` closes the nearest dialog or drawer, a backdrop click dismisses a dialog (which you can disable with `data-no-dismiss`), and clicking a link inside a drawer automatically closes it.
- **`menus(root)`**: handles `<details class="dropdown">` dropdowns. Where the Popover API is available, it upgrades the panel to the top layer (supporting light-dismiss, the Esc key, and exclusive opening) and positions it below the trigger (add `class="right"` on the `<details>` to right-align). Without the Popover API, it falls back to standard `<details>` toggle handling, which ensures only one dropdown is open at a time, closes the dropdown on click-outside or Esc, and closes it when an item is clicked.
- **`placePopover(panel, anchor, opts?)`**: positions a top-layer popover under an anchor element. Top-layer popovers (`[popover]`) escape all ancestor `clip-path` and `overflow` constraints, but the browser does not position them automatically. Call this function after `panel.showPopover()` (and again on scroll or resize while open) to align the panel under the `anchor` based on its viewport bounding rectangle. Available options are: `{ align = 'start' | 'end', gap = 6, matchWidth = false }` where `'end'` right-aligns, `gap` is the pixel offset below the anchor, and `matchWidth` sets the panel's minimum width to the anchor's width. This is the underlying primitive for `menus()`, and it is the recommended way to build an autocomplete `<ul popover class="listbox">` component.
- **`toolbars(root)`**: roving tabindex for `[role="toolbar"]` (providing one tab stop where arrow keys move focus and `Home`/`End` jump). Add `class="vertical"` on the toolbar to enable Up/Down arrow navigation.
- **`tooltips(root)`**: renders `[data-tip="text"]` on hover or focus using a single body-level `#vui-tooltip` element. It uses `position: fixed` so it is never clipped by an ancestor's `overflow` or `clip-path`. It also marks target elements with `.vui-tip-js` to suppress the CSS-only `::after` fallback.
- **Live clock (via `init`)**: writes `HH:MM:SS` into every element with `[data-vui-clock]` once per second.
- **`toast(message, options?)`**: programmatic snackbar that returns `{ el, dismiss }`. It automatically creates an `aria-live` `.toaster` region on its first invocation.
  - Options: `{ tone = '', title = '', duration = 4000, role = 'status', dismissible = true, placement = '' }`.
  - `tone`: `'info' | 'warn' | 'threat' | 'secure'` (also accepts a bare string shorthand: `toast('msg', 'secure')`).
  - `role`: `'status'` (polite) or `'alert'`.
  - `placement`: applied to the toaster region (e.g. `'top'`, `'start'`, `'center'`).
  - `duration: 0` keeps it until dismissed; `dismissible` adds a close button.

```js
import { init, toast, setValue } from 'vantaui-css/js';
init();
toast('Telemetry uplink restored', { tone: 'secure', title: 'ONLINE' });

// animate a gauge to 92 over 800ms
setValue(document.querySelector('.gauge'), 92, 800);
```

Default export: `VantaUI = { init, tabs, setValue, drawers, tooltips, menus, placePopover, toolbars, toast }`.

### Autocomplete with placePopover

```js
import { placePopover } from 'vantaui-css/js';
const input = document.querySelector('#names');
const list  = document.querySelector('#names-ac'); // <ul popover="manual" class="listbox">

function open() {
  list.showPopover();
  placePopover(list, input, { matchWidth: true }); // align under, match field width
}
// keep it pinned while the page moves
addEventListener('scroll', () => list.matches(':popover-open') && placePopover(list, input, { matchWidth: true }), true);
addEventListener('resize', () => list.matches(':popover-open') && placePopover(list, input, { matchWidth: true }));
input.addEventListener('blur', () => list.hidePopover());
```

The `<ul>` lives in the top layer, so it is never clipped by the chamfer/notch `clip-path` of a surrounding panel. No teleport-to-body logic is required.

## Vue 3

```js
import { createApp } from 'vue';
import VantaUI from 'vantaui-css/vue'; // imports CSS, adds .vui to <body>, boots behaviours
import App from './App.vue';
createApp(App).use(VantaUI).mount('#app');
// options: app.use(VantaUI, { behaviours: false, bodyClass: false, root: el })
```

## Nuxt 3

```ts
// nuxt.config.ts
export default defineNuxtConfig({
  modules: ['vantaui-css/nuxt'],
  VantaUI: { behaviours: true, bodyClass: true }, // both default true
});
```

The module registers the stylesheet, adds `.vui` to `<body>` app-wide, and (client-side) boots the optional behaviours.
